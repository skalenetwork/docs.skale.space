<Api openapi="api-reference/openapi.json" path="/" method="post" example="eth_newPendingTransactionFilter" />


Creates a filter in the node to notify when new pending transactions arrive.

## Request

```json
{
  "jsonrpc": "2.0",
  "method": "eth_newPendingTransactionFilter",
  "params": [],
  "id": 1
}
```

### Parameters

None.

## Response

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": "0x1"
}
```

### Response Types

- **result** (`string`): Filter ID as a hexadecimal string

## Example

<CodeGroup>

```bash Request
curl -X POST https://mainnet.skalenodes.com/v1/honorable-steel-rasalhague \
  -H "Content-Type: application/json" \
  -d '{

    "id": 1

```

```json Response
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": "0x1"
}
```

</CodeGroup>

## Notes

- Available on all node types
- Use the filter ID with [eth_getFilterChanges](/api-reference/methods/eth-getFilterChanges) to get pending transactions
- Pending transactions are those submitted but not yet included in a block
