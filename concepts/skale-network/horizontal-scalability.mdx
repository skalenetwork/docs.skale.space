## Overview

SKALE achieves infinite horizontal scalability by adding new SKALE Chains to the network. Unlike vertical scaling (increasing resources of existing chains), horizontal scaling adds capacity by adding new chains, enabling the network to scale linearly with demand.

## What is Horizontal Scalability?

Horizontal scalability means:

- **Add Capacity** - Add new chains to increase network capacity
- **Linear Scaling** - Capacity scales linearly with number of chains
- **No Limits** - No theoretical limit on number of chains
- **Independent Chains** - Each chain operates independently

## How SKALE Scales Horizontally

### Adding New Chains

SKALE scales by:

1. **Chain Creation** - New SKALE Chains are created through SKALE Manager
2. **Resource Allocation** - Resources are allocated to new chains
3. **Node Assignment** - Nodes are assigned to new chains
4. **Network Integration** - Chains integrate into the network
5. **Capacity Addition** - Network capacity increases

### Shared Validator Pool

The shared validator pool enables:

- **Efficient Scaling** - Validators can secure multiple chains
- **Resource Efficiency** - Better use of validator resources
- **Cost Effectiveness** - Lower costs for new chains
- **Network Security** - All chains benefit from network security

## Scaling Mechanisms

### Chain Creation

New chains are created through:

- **SKALE Manager** - Chain creation via SKALE Manager contracts
- **Resource Allocation** - Resources allocated per chain
- **Configuration** - Chain configuration and setup
- **Integration** - Integration into network

### Resource Management

Resources are managed through:

- **Allocation** - Resources allocated per chain
- **Virtualization** - Resource virtualization for efficiency
- **Containerization** - Container-based resource management
- **Orchestration** - Resource orchestration and management

### Network Growth

Network grows through:

- **Chain Addition** - Adding new chains to network
- **Validator Growth** - Growing validator pool
- **Infrastructure Expansion** - Expanding infrastructure
- **Capacity Planning** - Planning for capacity needs

## Comparison to Vertical Scaling

### Vertical Scaling

Traditional vertical scaling:

- **Resource Increase** - Increase resources of existing chains
- **Hardware Limits** - Limited by hardware capabilities
- **Diminishing Returns** - Diminishing returns at scale
- **Cost Increase** - Costs increase with resource needs

### Horizontal Scaling

SKALE's horizontal scaling:

- **Chain Addition** - Add new chains for capacity
- **No Hardware Limits** - No hardware-based limits
- **Linear Returns** - Linear returns at scale
- **Cost Efficiency** - Cost-efficient scaling
