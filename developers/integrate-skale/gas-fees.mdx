SKALE offers several models for handling gas, all designed to eliminate friction for end users. Which model fits depends on your chain type and deployment.

## The Models

### Zero Gas Fees (sFUEL)

The original model — every SKALE Chain has a native gas token (sFUEL) that has **no monetary value**. It exists for EVM compatibility so wallets, tools, and SDKs work out of the box. sFUEL is pre-minted in massive surplus and recycled onchain — it never runs out.

```
User sends tx → consumes sFUEL (valueless) → tx executes
              → sFUEL recycled back to distribution pool
```

<Note>
Developers and users can acquire sFUEL at https://sfuelstation.com
</Note>

### Compute Credits (Credit Chains)

For SKALE Expand deployments, compute credits replace sFUEL with a directly purchasable resource. The chain owner **buys credits** in a simple subscription, and users transact for free — no tokens, no sFUEL, no friction.

**How buying credits works:**

1. Go to **[https://portal.skale.space/chains](https://portal.skale.space/chains)**
2. Select your chain → **Manage** → **Connect Wallet**
3. Choose months to prepay (up to 24)
4. Bridge SKL and confirm payment

```
Chain owner buys credits → credits deposited in chain pool
                        → users send tx → compute deducted from pool
                        → users never pay or hold tokens
```

### Gasless Transactions (Proof-of-Work)

Gasless transactions let users send transactions with **zero sFUEL and zero credits**. A Proof-of-Work algorithm generates a magic number that validates the transaction instead of any gas payment. Best for maximally decentralized onboarding.

```
Client computes PoW → generates magic number
                     → sends tx with magic number as gasPrice
                     → validator verifies work → tx executes
```

<Note>
Proof-of-Work is disabled by default on credit chains.
</Note>

### Gas Chains

A gas chain uses a real token (SKL, ETH, USDC, or any ERC-20) as the gas token. Users pay per-transaction in that token, similar to an L2. Available on SKALE Expand deployments.

## Which Model to Use

| Model | Who Pays | User Needs Tokens? | Best For |
|-------|----------|-------------------|----------|
| Zero Gas Fees (sFUEL) | Nobody | sFUEL (free) | SKALE on Ethereum chains |
| Compute Credits | Chain owner | Nothing | Consumer apps, games |
| Gasless (PoW) | Nobody | Nothing | Frictionless onboarding |
| Gas Chain | Users | Gas token | DeFi, financial apps |
